# Аудит бэкенда и исправления

## Что я исправил

### Поток авторизации / профиля

* Сохранил session-based auth bridge через `SessionAuthenticationFilter`.
* Оставил вход по **никнейму или email** в `HomeController.findUserByLogin(...)`.
* Сохранил редиректы на `/profile`, потому что эта страница теперь корректно открывается.
* `ProfileController` больше не ведёт на несуществующие шаблоны для `/my/tournaments`, `/my/matches`, `/notifications`; теперь эти маршруты перенаправляют на существующие страницы.

### Бэкенд пользователя

* Добавил `createdAt` в `UserProfileDTO`, чтобы страница профиля могла показывать дату регистрации.
* Расширил `UserService`:

  * создание пользователя
  * обновление собственного профиля
  * обновление текущего пользователя без явного `id`
  * обновление/сброс аватарки
  * смена пароля
* Добавил `ChangePasswordDTO`.
* Добавил совместимые endpoints в `UserApiController`, которые использует текущий фронт:

  * `PUT /api/users/update`
  * `POST /api/users/avatar`
  * `DELETE /api/users/avatar`
  * `POST /api/users/change-password`
* Сохранил существующие REST endpoints:

  * `GET /api/users/{id}`
  * `POST /api/users`
  * `PUT /api/users/{id}`

### Бэкенд команды

* Создание команды теперь требует `gameTypeId` и получает `GameType` из базы данных.
* Ограничение размера команды теперь учитывается через `GameType.maxPlayers`.
* В `TeamFullDTO` добавлены:

  * `currentMembersCount`
  * `maxMembersCount`
* Принятие приглашения и прямое добавление участника теперь тоже проверяют лимит команды.

### Согласованность уведомлений

* Привёл тип приглашения к единому виду `TEAM_INVITE` в бэкенд-логике.
* Обновил `schema.sql`, теперь тип уведомления по умолчанию — `TEAM_INVITE`.

### Совместимость фронта / профиля

* `profile.js` обновлён под текущий бэкенд DTO/JSON-ответы:

  * использует `createdAt`
  * использует `/api/users/update`
  * работает с endpoints для аватара и смены пароля
  * больше не зависит от отсутствующих полей `bio`, `rating`, `totalWins`, `totalTournaments`
* `login.html` очищен от дублирующихся подключений CSS/JS.

### Справочник по DTO

* Переписал `frontend-dfh-guide.md` в полноценный **справочник по DTO** с актуальными именами DTO и endpoint-ами.

## Важные замечания

* Я не смог выполнить полную Maven-сборку в контейнере, потому что wrapper требует доступ в сеть для скачивания Maven.
* Самые важные несоответствия бэкенда были исправлены за счёт синхронизации DTO, user endpoints, логики страницы профиля и ограничений по размеру команды.
